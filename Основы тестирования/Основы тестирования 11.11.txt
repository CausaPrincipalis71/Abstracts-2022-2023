Процесс
У процесса есть входные данные, для его работы.
В этот процесс должен поступить какой-то объект, чтобы он сам преобразовался, или с его помощью были преобразованы другие объекы
Есть вход, и есть выход
Выход тоже является объектом (услуга, программа,)
Чтобы это преобразование произошло, нужны механизмы.
          Управление
              |
              |
              |
              V
        -------------
Вход----|---Процесс--|---->Выход
        -------------
              ^
              |
              |
           Механизмы

Для корректной работы процесса важны все факторы. Для преобразования "Входа" в "Выход" должен быть вход, управление и механизмы.
Мы что-то делаем, чтобы оно что-то преобразовало во что-то, при помощи чего-то при управлении чем-то

На лабе мы обнаружим много папочек.
Там находятся .pdf файлы. В этих файлах описан функционал программы. Прочитав все документы, которые есть в папке, не надо программировать.
Задача в том, чтобы выделить процессы, в рамках этого процесса выделить отдельные действия(варианты использования, в терминологии)
Попробовать разложить вариант использования на элементарные составляющие, по шаблону
Этапы реализации, альтернативный вариант использования (если что-то произойдёт не так в нормальном способе, есть ли возможность получить конечный результат другим способом)
Альтернативных вариантов может не оказаться.
Входные условия: те параметры, которые позволяют реализовать нормальный вариант.
Исключения: что может произойти не так, во время выполнения операций
